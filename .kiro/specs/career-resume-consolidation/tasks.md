# Implementation Plan

- [x] 1. Foundation: データ定義とスキーマの設定
- [x] 1.1 経歴データ定義 of 型拡張
  - 経歴データ（会社別の複数プロジェクト、スキルカテゴリ、自己PRなど）を表現するためのTypeScript型定義を新設・拡張する
  - 完了時に、データ型定義のコンパイルエラー（`bun typecheck`）が発生せず、新しい構造の型安全性が検証できていること
  - _Requirements: 2.1, 2.3_

- [x] 1.2 匿名化された経歴データの配置
  - 実在する企業名や商標（menu株式会社、PlayStation、Blu-rayなど）を一切含まない、完全に一般化された最新の職歴情報および自己紹介・自己PRデータをJSON形式で配置する
  - 完了時に、配置されたデータが新設した経歴データ型定義をすべて満たし、インポートエラーなく読み込めること
  - _Requirements: 3.1, 3.2_

- [x] 2. Core: ナビゲーション整理と旧ルートのクリーンアップ
- [x] 2.1 (P) ナビゲーションメニューの更新
  - ヘッダー内のナビゲーションリンク定義から古い職務経歴ページ（`/career`）へのリンクを削除し、「経歴・実績」への導線に整理する
  - 完了時に、画面上のヘッダー表示から「職務経歴」のリンクが消え、「経歴・実績」のみが表示されていること
  - _Requirements: 1.1_
  - _Boundary: Header_

- [x] 2.2 (P) 旧ルートファイルの削除
  - 古い経歴ページ（`/career`）のルート定義ファイルを削除し、不要になったルートの参照をホームページリンク等から排除する
  - 完了時に、`/career` へのアクセス時にNotFoundになり、ホームページの「経歴・実績を見る」ボタンから正しく `/resume` に遷移できること
  - _Requirements: 1.2_
  - _Boundary: Route(/career)_

- [x] 3. Core: UIコンポーネントの作成と改善
- [x] 3.1 タイムライン描画の拡張
  - タイムライン表示コンポーネントを修正し、会社ごとの中に複数のサブプロジェクト（プロジェクト名、期間、詳細箇条書き）を階層的にループ描画できるようにする
  - 完了時に、画面上にECサービス運営企業での「Web版リプレイス開発」や「モバイルアプリ開発」などの期間別実績が階層的に表示され、それぞれで使用された技術スタックが明記されていること
  - _Requirements: 2.2, 2.3_
  - _Boundary: Timeline_

- [x] 3.2 統合ビューのレイアウト修正
  - 経歴・実績表示用のメインコンポーネントを修正し、新しく拡張されたデータ（職務要約、活かせる経験・技術、自己PR）を適切なセクションとデザインで表示する
  - 完了時に、`/resume` ページ上に「職務要約」「スキルセット」「自己PR」「職務経歴タイムライン」がデザインシステムに従って調和してレイアウト表示されていること
  - _Requirements: 1.3, 2.1_
  - _Boundary: ResumeView_

- [x] 4. Validation: ビルドおよび動作検証
- [x] 4.1 全体ビルドおよびSSGクロール検証
  - ビルドスクリプトを実行し、静的な型チェック、静的検証（Linter）、および静的HTML書き出し（プリレンダリング）が正常に完了することを確認する
  - 完了時に、全ページが静的HTMLとしてエラーなく出力され、`bun typecheck` や Biome が警告なしでパスすること
  - _Requirements: 4.1, 4.2_

- [x] 4.2 データのプライバシー整合性検証
  - Gitリポジトリのインデックスおよびソースコード全域において、実在する企業名や非公開製品名・商標の文字列が含まれていないかを検証する
  - 完了時に、リポジトリ内のコミット可能ファイルに機密性の高い固有名詞が一切残っていないことが確認できていること
  - _Requirements: 3.2_
