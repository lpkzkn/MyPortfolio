# Requirements Document: component-catalog

## Introduction
本仕様は、ポートフォリオサイト内の `/catalog` ルートに、自作デザインシステムコンポーネント（`src/components/ui/` 配下）の動作を一覧・確認できるインタラクティブなカタログページを構築する機能を定義します。ソースコードを読んだエンジニアが、UIコンポーネントの品質とバリエーションをブラウザ上で直接体験できるショーケースとして機能させます。

## Boundary Context
- **In scope**:
  - `/catalog` ルートおよびカタログ画面の構築
  - `src/features/catalog/` 配下への実装コロケーション
  - `src/components/ui/` 配下の共通コンポーネントのインタラクティブなデモ（全 Variant の表示）
  - Tailwind v4 トークン（カラー、フォント、スペーシング）のビジュアルリファレンス表示
  - アクセシビリティ対応（キーボード操作、スクリーンリーダー対応）の動作デモ
- **Out of scope**:
  - Storybook 等の外部ツールを使ったコンポーネントカタログの生成
  - コンポーネント自体の実装（`src/components/ui/` は infrastructure スペックの所有物）

## Requirements

### Requirement 1: コンポーネントデモの網羅的な表示
**Objective:** ポートフォリオを見るエンジニアとして、`/catalog` ページにアクセスするだけで、このプロジェクトで定義されている全ての共通 UI コンポーネント（Button, Card, Badge 等）がどのように動作するか（バリエーション・状態遷移）をインタラクティブに確認したい。

#### Acceptance Criteria
1. `/catalog` にアクセスした際、`src/components/ui/` に実装されている全コンポーネントのデモセクションが一覧で表示されること。
2. 各コンポーネントに対して、利用可能な全バリアント（例: Button の `primary`, `secondary`, `ghost`）と状態（`hover`, `active`, `disabled`, `focus`）が実際の動作するコンポーネントとして描画されること。
3. 各デモセクションには、そのコンポーネントの使用例コードスニペット（読み取り専用）が表示されること。

### Requirement 2: デザインシステムトークンのビジュアルリファレンス
**Objective:** ポートフォリオを見るエンジニアとして、このプロジェクトが採用しているデザインシステムのカラーパレットやフォントが一覧で視覚的に確認できることで、設計の一貫性と品質の高さを理解したい。

#### Acceptance Criteria
1. カタログページ内の「デザイントークン」セクションに、Tailwind v4 で定義したカラーパレット（CSS Variables）が色見本とその変数名（例: `--color-primary`）と共に表示されること。
2. ダークモードとライトモードを切り替えた際に、カラーパレットの表示も連動して変化すること。
3. 使用中のタイポグラフィスケール（font-size の段階）が、サンプルテキストと共に一覧表示されること。
