# Requirements Document: github-showcase

## Introduction
本仕様は、GitHub の公開 API を利用して、ポートフォリオ訪問者にリアルタイムな開発活動（リポジトリ一覧、コミット履歴など）を動的に表示する機能を定義します。静的ホスティング（GitHub Pages）環境下における、クライアントサイドフェッチの設計美と堅牢なエラーハンドリングをアピールします。

## Boundary Context
- **In scope**:
  - GitHub REST API（パブリックデータのみ）へのクライアントサイドフェッチ処理
  - TanStack Query を活用したデータキャッシュと stale-time 制御
  - ローディング中のアニメーションつき Skeleton UI の実装
  - APIエラー・レートリミット時の適切なフォールバック表示
  - ポートフォリオ内 GitHub ショーケースページの UI（リポジトリカード、コントリビューション表示等）
- **Out of scope**:
  - プライベートリポジトリへのアクセス・OAuth認証処理
  - サーバーサイドでの GitHub API プロキシ

## Requirements

### Requirement 1: GitHub API 連携とクライアントキャッシュ
**Objective:** 開発者・採用担当者として、ポートフォリオを訪れた際に最新のリポジトリ一覧が表示されることで、現在どんな技術に取り組んでいるかをすぐに把握できるようにしたい。

#### Acceptance Criteria
1. `/github` もしくはポートフォリオトップ上のセクションとして、GitHub API から取得した自身のパブリックリポジトリ一覧（スター数、言語、最終更新日等の情報を含む）が表示されること。
2. TanStack Query の `useQuery` によりデータがキャッシュされ、同一セッション内での再フェッチ回数が抑制されること（`staleTime` を明示的に設定すること）。
3. APIレートリミット（403、HTTP 429等）が発生した際、ユーザーに分かりやすいフォールバックメッセージ（例：「現在データを取得できません。後ほどご確認ください。」）が表示されること。

### Requirement 2: ローディング体験とアニメーション（Skeleton UI）
**Objective:** 訪問者として、データ取得中も白い空白ではなく、なめらかなスケルトンアニメーションで「読み込み中」が伝わる、快適なローディング体験を得たい。

#### Acceptance Criteria
1. GitHub API フェッチ中（ローディング状態）において、コンテンツのレイアウトを模したスケルトンスクリーン（shimmer アニメーション付き）が表示されること。
2. データ取得完了後、スケルトンからコンテンツへ滑らかなフェードイン/アニメーションで遷移すること。
3. データ取得エラー時には、スケルトンの代わりに明示的なエラーメッセージ UI（再試行ボタン付き）が表示されること。
