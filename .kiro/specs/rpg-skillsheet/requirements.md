# Requirements Document

## Introduction
RPG風スキルステータスとアニメーション付きレーダーチャートの表示。

【解決したい課題と背景】
ポートフォリオサイトのスキルシートにおいて、単なるテキストの羅列だけではなく、閲覧者が楽しめ、かつ開発者の得意技術や強みのバランスが直感的に伝わるビジュアル表現を導入したい。

【変更方針】
1. **RPG風パラメータ表示**:
   - ゲームのステータス画面をイメージした遊び心のあるレイアウトを採用する。
2. **アニメーション付きレーダーチャート**:
   - スキルの得意領域のバランス（フロントエンド、バックエンド、インフラ、プロセスなど）を表現するレーダーチャートを配置する。
   - ページ表示（ビューポート侵入）時に、中心から値が滑らかに拡大するアニメーションを伴う。
3. **データ匿名化（一般化）の維持**:
   - スキルデータや項目記述においても、実在する製品・団体名等は伏せる（ECサービス、家庭用ゲーム機など）。
4. **ドリルダウン形式の階層表示**:
   - 大項目（フロントエンドなど）から、中項目（アプリ、ゲームなど）、小項目（言語、CI、E2Eなど）へとクリックで深掘りできるインタラクティブなUIを提供する。

## Boundary Context (Optional)
- **In scope**: RPG風の視覚的なスキルシートレイアウト、レーダーチャートのアニメーション表示、クリックによるスキルのドリルダウン（階層化）表示、スキルの一般化（匿名化）されたデータの表示。
- **Out of scope**: スキルデータを管理・編集するバックエンドシステムやCMSの構築（データは静的に定義される想定）。
- **Adjacent expectations**: 既存のポートフォリオサイトのカラースキームやデザインシステムと調和しつつ、RPG風の特有のアクセントを持たせること。

## Requirements

### Requirement 1: RPG風パラメータ表示
**Objective:** As a 閲覧者, I want RPG風のステータス画面をイメージした遊び心のあるレイアウトを見ること, so that 開発者のスキルや強みを楽しみながら確認できる

#### Acceptance Criteria
1. When スキルシートのページが読み込まれたとき, the system shall RPG風のパラメータレイアウトでスキルデータを表示する
2. The system shall データの匿名化を維持するため、実在する製品名や団体名を伏せて表示する

### Requirement 2: アニメーション付きレーダーチャート
**Objective:** As a 閲覧者, I want 開発者のスキルのバランスを示すアニメーション付きレーダーチャートを見ること, so that 得意領域のバランスを直感的に把握できる

#### Acceptance Criteria
1. When レーダーチャートがビューポートに侵入したとき, the system shall 中心から値が滑らかに拡大するアニメーションを伴ってチャートを描画する
2. The system shall レーダーチャートの軸に大項目（フロントエンド、バックエンド、インフラ、プロセスなど）を配置する

### Requirement 3: インタラクティブなスキルのドリルダウン表示
**Objective:** As a 閲覧者, I want 大項目をクリックして詳細な中項目や特定の技術を見ること, so that 開発者の専門性をより深く探求できる

#### Acceptance Criteria
1. When ユーザーが大項目（例：フロントエンド）をクリックしたとき, the system shall 展開してその中項目（例：Web App、Game）を表示する
2. When ユーザーが中項目をクリックしたとき, the system shall 具体的な技術やスキル（例：言語、CI/CD、E2E）を表示する
3. If すでに展開されている項目をユーザーが再度クリックしたとき, the system shall その配下の項目を折りたたんで非表示にする
